# 老斯密碼 - Streamlit 版

這是把原本 Tkinter 桌面程式（`gui_app.py`）改寫成的 Streamlit 網站版本。
**商業邏輯完全沒有改動** —— `teacher_translator.py`（資料庫存取、翻譯比對、投票、徽章、審核…等所有規則）原封不動沿用，
只有「畫面」從 Tkinter 換成 Streamlit（`app.py`）。

## 檔案說明

| 檔案 | 說明 |
|---|---|
| `app.py` | Streamlit 介面入口，**啟動這個檔案** |
| `teacher_translator.py` | 原有商業邏輯 (未更動) |
| `Goldsent.py` | 原專案的金句生成器類別 (保留，目前未被直接呼叫) |
| `laosimida.db` | 主要資料庫 (語句、票數、使用者、投稿) |
| `sentences.db` | 大師金句資料庫 |
| `requirements.txt` | Python 套件需求 |

## 本機執行步驟

1. 安裝 Python 3.9 以上版本。
2. 在這個資料夾打開終端機，安裝套件：
   ```bash
   pip install -r requirements.txt
   ```
3. 啟動網站：
   ```bash
   streamlit run app.py
   ```
4. 瀏覽器會自動打開 `http://localhost:8501`，看到登入畫面即代表成功。

## 部署到網路上（免費方式：Streamlit Community Cloud）

1. 把這個資料夾整個推上 GitHub（建一個新的 repo）。
2. 到 https://share.streamlit.io 用 GitHub 帳號登入。
3. 選擇剛剛的 repo，Main file path 填 `app.py`，按下部署。
4. 幾分鐘後就會拿到一個公開網址，例如 `https://你的專案.streamlit.app`。

## ⚠️ 關於資料庫的重要提醒

`laosimida.db` / `sentences.db` 是本機的 SQLite 檔案：

- **本機執行**：資料會正常寫入檔案、重開程式也會保留，跟原本 Tkinter 版的行為完全一樣。
- **部署到 Streamlit Community Cloud（或大部分免費雲端平台）**：這些平台的檔案系統是「暫時性」的，
  App 休眠、重啟或重新部署時，SQLite 檔案會被還原成 GitHub 上的版本，
  使用者投稿、投票、累積次數等「新增的資料」**不會被永久保留**。
  如果之後想讓資料真正長期保存，建議：
  - 換成雲端資料庫（例如 Supabase / Neon 的 PostgreSQL、Turso 的 SQLite-as-a-service），或
  - 部署在自己有持久化硬碟的伺服器 / VPS 上。

  目前這個版本適合：本機使用、小組內部 Demo、教學展示。

## 與原 Tkinter 版的對應關係

| Tkinter 版功能 | Streamlit 版 |
|---|---|
| 視窗頂部導覽列 | 左側 Sidebar 導覽按鈕 |
| 彈出視窗 (Toplevel) | 同一頁面切換內容區塊（透過 `st.session_state.view`） |
| `messagebox` 詢問確認 | 改成按鈕 + `st.info` / `st.warning` 提示 |
| `simpledialog` 輸入管理員密碼 | `st.text_input(type="password")` 表單 |
